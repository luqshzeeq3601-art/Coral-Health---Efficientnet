# EfficientNetB0 Coral — V4 Robust Model

This folder contains the **production model** used by the web application.  
Architecture: EfficientNetB0 + SWA (Stochastic Weight Averaging) robust ensemble.  
Canonical benchmark in this repo snapshot: **98.11%** held-out test accuracy on 159 test images using **224x224 single-scale ensemble inference**.

---

## Folder Structure

```
efficientnetb0_coral/
├── train_v4_robust.py       ← Main training script
├── split_info_v3.json       ← Optional cached split file (not present in every repo snapshot)
├── temperature.txt          ← Temperature scaling value (T=0.441) for calibration
│
├── models/                  ← Trained model weights (used by webapp)
├── outputs/                 ← Training evaluation metrics and plots
├── audit/                   ← Per-image audit to verify model performance
└── analysis/                ← Post-training XAI and visualisation tools
```

---

## Files

### `train_v4_robust.py`
**Run this to train or re-train the model.**  
Trains 5 seeds (42–46) of EfficientNetB0 with:
- Hard-example oversampling (×20) for difficult Bleached/Dead images
- SWA (Stochastic Weight Averaging) for better generalisation
- Benchmark reporting standardized around 224x224 single-scale ensemble inference

```bash
python train_v4_robust.py
```

Saves trained weights to `models/` and all evaluation outputs to `outputs/`.

---

### `split_info_v3.json`
Optional deterministic train/val/test split cache.  
If the file is absent, the training notebook and training script can reconstruct the same split logic using `random_state=42` and stratified splitting.

---

### `temperature.txt`
Temperature scaling calibration value (`T = 0.441`).  
Applied by the web app after ensemble averaging to improve probability calibration.  
Generated by `02_Modelling/scripts/calculate_temperature.py`.

---

## Accuracy note

Different inference protocols in this project snapshot can produce different accuracy values. To avoid mixing metrics across unlike-for-like evaluations, the modelling notebook treats **98.11%** as the canonical academic benchmark because it matches the stored classification report and confusion-matrix artefacts under the documented 224x224 single-scale ensemble setup.

## Preprocessing Standard

All scripts in this folder — training, audit, and analysis — use **the same OpenCV pipeline** to ensure identical results:

```python
img = cv2.imread(path)
img = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)
img = cv2.resize(img, (224, 224))   # INTER_LINEAR (default)
```

The web application (`04_Web_Application/app.py`) uses this same pipeline for inference.

## Do I Need to Re-Run Training?

| Situation | Action |
|---|---|
| Models in `models/` already exist | **No** — do not re-train |
| You want to verify per-image results | Run `audit/reproduce_canonical_results.py` |
| You changed training data or hyperparameters | Re-run `train_v4_robust.py` |
| Outputs look stale after re-training | Outputs are regenerated automatically by the training script |
